Basic

  • git config --global user.name Kateryna Khudieieva - необхідний для задання імені
  • git config--global user.email sadamochka@gmail.com - неоюхідний для задання електронної пошти
  • git init - команда для ініціалізації репозиторію
  • git status - команда для отримання поточного статусу файлів
  • git add - додання файлів в репозиторій
  • git commit - створення коментаря про внесені зміни
  • git log - команда для отримання історії комітів
  • git checkout - команда для перемикання або створення гілок
  • git branch - команда призначеня для того щоб дати назву гілці розробки
  • git push - команда для загрузки наших файлів на попередньо доданий репозиторій(відбуваються процеси запису та зжимання файлів)
  • git pull - це запит до репозиторію для додавання нової гілки розробки до основної гілки
  • git merge - це процедура яка виконується у репозиторію коли ми підтверджуємо склеювання гілки розробки з основною гілкою
  • git clone - команда для клонування файлів
  • gitignore файл - це файл який ми створюємо в корені репозиторію, щоб надалі добавити туди файли які Гіт буде ігнорувати при фіксуванні

Advanced

  • git revert - метод для безпечного скасування змін,зручна тим що не втрачаються коміти та не втрачається історія комітів, а просто створить новий коміт із скасуванням змін
  • git reset - команда для скасування локальних змін стану сховища Git(git revert є безпечнішим)
  • git rebase - Перебазування, схоже на merge, ви можете взяти всі коміти однієї гілки та передати їх на іншу в тому ж порядку, але не можна застосовувати для змін які вже були кудись відправленні
  • git cherry-pick схоже до перебазування , але вибір комітів робиться вручну, а не автоматично як в rebase
  • git stash - використовується якщо вам необхідно зробити щось інше а цю частину, що ви робили лишити на потім.Зміни які ви задали зберігаються в Стеш до якого ви можете згодом вернутись
